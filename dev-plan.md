# Autonomous Research Stack - Development Plan

> Synthesized from: Karpathy autorearch, Ouroboros, Sakana AI Scientist, SelfAI, Epsilon, DUMP, Actor-Curator, Curriculus library, and industry research.

---

## Project Vision

Build a production-ready autonomous research stack that enables an AI agent to continuously improve LLM training through an automated feedback loop—optimizing for a clear metric (val_bpb) while maintaining stability and reproducibility.

**Core Philosophy:**
- Single-GPU focused (like Karpathy autorearch)
- val_bpb as the universal metric
- Bounded autonomy with human oversight
- Maximum improvement per experiment, not experimentation volume
- Build incrementally, ship at each phase

### Architectural Insights

1. **Karpathy autorearch** - The foundational pattern:
   - Three files: `program.md` (goals), `train.py` (editable), `prepare.py` (fixed evaluator)
   - val_bpb (bits per byte) as the universal metric
   - 5-minute fixed budget per experiment
   - Agent edits one file, commits on improvement, reverts on regression

2. **Ouroboros** - Meta-loop advancement:
   - Each generation rewrites research methodology itself
   - Population tournament mode (multiple competing lineages)
   - Divergence-aware selection across legibility, coherence, ambition

3. **AI Scientist** - Research automation:
   - Full pipeline: idea → experiment → analysis → manuscript
   - First AI-generated peer-reviewed paper accepted at ICLR workshop

4. **Epsilon** - Scientific rigor:
   - Structured gap discovery
   - Self-correcting discovery loops
   - p-hacking prevention via architectural constraints

5. **DUMP** - Intelligent curriculum:
   - Advantage-based learnability scoring
   - UCB-based distribution selection
   - Automated curriculum for RL post-training

## Implementation Notes

### Phase 1 Priority: Making Synthetic Real

The highest-priority first step is making `synthetic_data.py` actually call an LLM:

```python
# Pseudocode for LLM-based generation
def generate_synthetic(n=200, model="claude-sonnet-4-20250514"):
    client = anthropic.Client()
    prompts = []
    for _ in range(n):
        response = client.messages.create(
            model=model,
            max_tokens=1024,
            messages=[{
                "role": "user",
                "content": "Generate a diverse training prompt about ML systems optimization."
            }]
        )
        prompts.append(response.content[0].text)
    return quality_filter(prompts)
```

### Configuration Management

Use environment variables for configuration:

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
_experimentBudget=500  # max experiments per run
_valTarget=0.95  # target val_bpb
```

```sql
CREATE TABLE experiments (

    val_bpb_after REAL,
    status TEXT,  -- 'kept', 'reverted'
    failure_classification TEXT,
    git_commit TEXT
);
```
